## Korvac entity LIFECYCLE — the scheme OSCILLATES (Saga ⇄ Revealed) every Twist; "Korvac" Villain exists ONLY while "Korvac Revealed" (Side B) is face up

- **THE RULING (implementation fork):** Korvac is a 19-Attack/9VP "Villain" that exists ONLY while the **Korvac Revealed** side (Side B) is face up. The scheme genuinely TOGGLES back and forth — Side A "Saga" Twist transforms to B; Side B "Revealed" Twists (2,4,6) transform back to A; etc. So Korvac is added/removed in sync with the active side each transform: implementation choice **(b)**, NOT (a) "placed once on reveal and persists." While "The Korvac Saga" (Side A) is face up, Korvac is NOT a fightable Villain.
- **WHY — the transform rule + Twist numbering:**
  - General rule (Revelations rulesheet p.2, "Double-Sided Transforming Schemes", verbatim): *"In keeping with the Revelations theme, all the Schemes in this set are double-sided 'Transforming Schemes.' Start with the side face up that says 'Setup.' Whenever it tells you to 'Transform this Scheme,' flip it over. **Use only the rules showing on the side currently face up.**"* The final sentence is decisive: when Side A is face up, ONLY Side A's rules apply — Side B's "This Scheme counts as a 19 Attack 'Korvac' Villain" is NOT in effect, so there is no fightable Korvac while Saga is showing.
  - Twist text (inventory verbatim, docs/card-inventory/final/revelations.md lines 684–691):
    - Side A **The Korvac Saga** — Setup: 8 Twists. Twist: *"Each player must discard down to four cards or KO a Bystander from their Victory Pile to 'search for the Korvac Entity.' This Scheme Transforms."* (→ flips to B)
    - Side B **Korvac Revealed** — Special Rules: *"This Scheme counts as a 19 Attack 'Korvac' Villain worth 9VP. If you defeat Korvac, KO the Mastermind and all its Tactics."* **Twist 2, 4, 6:** *"Each player discards an Avengers Hero or gains a Wound. This Scheme Transforms."* (→ flips back to A) **Twist 8:** *"Evil Wins!"*
  - **Twist sequence walk (the disambiguator is the per-side Twist NUMBERING — Saga's Twist has no number / is the catch-all "the Saga Twist"; Revealed's are explicitly 2,4,6,8 by Twist-count):**
    - Start: Side A (Saga) face up. Korvac NOT present.
    - **Twist 1** is read off the FACE-UP side = Saga's Twist → "...This Scheme Transforms." Flip to B (Revealed). Korvac NOW present/fightable (1st reveal).
    - **Twist 2** read off Revealed = its "Twist 2,4,6" line → "...This Scheme Transforms." Flip back to A (Saga). Korvac GONE.
    - **Twist 3** read off Saga = Saga Twist → Transforms. Flip to B. Korvac present again (2nd reveal).
    - **Twist 4** read off Revealed = "Twist 2,4,6" → Transforms. Flip to A. Korvac gone.
    - **Twist 5** (Saga) → Transforms → B. Korvac present (3rd reveal).
    - **Twist 6** (Revealed, "2,4,6") → Transforms → A. Korvac gone.
    - **Twist 7** (Saga) → Transforms → B. Korvac present (4th reveal). Side B is face up.
    - **Twist 8** read off Revealed = its **"Twist 8: Evil Wins!"** → game over (Evil) UNLESS Korvac was defeated earlier.
  - This is why Side B numbers ONLY the even Twists (2,4,6,8): the scheme alternates sides, so the EVEN Twists land while Revealed is showing and the ODD Twists land while Saga is showing (Saga's single un-numbered Twist covers all odd-count Twists). The mechanism REQUIRES a genuine back-and-forth toggle — a one-way "reveal once and stay" would never produce a Twist 8 on the Revealed side via this alternation, and would leave the Saga Twist firing only once. So the code's "transform BACK as well (Saga ⇄ Revealed)" is CORRECT; it does NOT transform one way and stay.
- **ANSWERS TO THE THREE QUESTIONS:**
  1. Korvac is fightable ONLY while Side B (Revealed) is face up. A later Twist transforming BACK to Saga (Side A) removes Korvac as a fightable Villain until it reveals again ("use only the rules showing on the side currently face up", rulesheet p.2). → choice **(b)** add/remove in sync with the active side.
  2. The reveal is NOT one-way. It is a genuine back-and-forth TOGGLE driven by the Twists (Saga Twist → Revealed; Revealed Twist 2/4/6 → Saga).
  3. The scheme transforms BOTH directions, alternating every Twist. It does NOT only transform Saga→Revealed and stay. Twist 8 (read off the Revealed side, where Twist 7 left it) is "Evil Wins!" — the deadline to have defeated Korvac.
- **WINDOW-OF-VULNERABILITY NOTE (gameplay consequence):** Korvac can only be DEFEATED during an odd-Twist window (after Twists 1, 3, 5, or 7) while Revealed is up. You must build Attack ≥19 and land the defeat in a turn when Side B is face up, before Twist 8. If you don't defeat him in a given Revealed window, the next Twist hides him again until the following odd Twist.

The Twists say "each player" (not "each OTHER player"), so in 1-player solo you simply do them yourself (no need to invoke What If? p.24 "each other player → do it yourself" or the Golden Solo other-player FAQ). Whether Korvac is present is purely "which side is face up" — same in both modes. (Win-on-defeat handling and the Final-Blow bypass are the SEPARATE entry above.)

## Ultimate Nullifier (Mr. Fantastic, FF) does NOT rules-correctly negate Korvac's "If you defeat Korvac, KO the Mastermind" WIN-trigger — that consequence is a SCHEME on-defeat rule, NOT a "Fight effect"

- **THE RULING (solo, both modes):** The Ultimate Nullifier CANNOT (rules-correctly) cancel Korvac's "If you defeat Korvac, KO the Mastermind and all its Tactics" win-trigger. The Korvac win-trigger is a **Scheme Special Rule that triggers ON DEFEAT** of the Korvac scheme-Villain — it is NOT a "Fight effect" of the kind the Ultimate Nullifier targets. The code's current behavior (Nullifier negates the win-trigger because Korvac's defeat→KO-Mastermind is wired as a Location `fightEffect`) is a side effect of the technical Location-reuse build, NOT a faithful rules outcome. **Recommendation: EXEMPT the Korvac win-trigger from the Ultimate Nullifier.** (Practically harmless either way because the Nullifier is OPTIONAL — see below — but the rules-correct outcome is exemption.)
- **THE THREE DISTINCTIONS (why it's not a Fight effect):**
  1. **SOURCE.** Ultimate Nullifier (inventory verbatim, docs/card-inventory/final/fantastic-four.md line 190): *"If an enemy you fight this turn would have a Fight effect, you may cancel that effect instead."* Scope = a **Fight effect on an enemy you fight** (i.e. text printed on a Villain/Henchman/Mastermind card). Korvac's win-trigger is printed on the **SCHEME** ("Special Rules" line of Korvac Revealed, Side B), not on an enemy card. The "19 Attack 'Korvac' Villain" is a Scheme that *counts as* a Villain for fighting; the KO-Mastermind clause is the Scheme's own Special Rule, not an enemy-card Fight effect.
  2. **TRIGGER TIMING.** Core "How to Fight a Villain" (Core_Set.pdf p.13, verbatim): *"If the Villain card has a 'Fight' effect on it, do what it says prior to comparing Attack totals."* A Fight effect fires **WHEN YOU FIGHT, BEFORE comparing Attack** — it resolves whether or not you actually defeat the enemy. Korvac's clause is gated on *"If you DEFEAT Korvac"* — it fires **AFTER** comparing Attack and winning. An on-defeat consequence is categorically a different timing window from a "prior to comparing Attack totals" Fight effect. The Ultimate Nullifier intercepts the pre-comparison Fight-effect window; the win-trigger is not in that window.
  3. **NATURE.** A "Fight effect" is a discrete listed effect on the card you punch through (Core p.13 bullet "If the Villain card has a 'Fight' effect on it"). Korvac's "KO the Mastermind and all its Tactics" is the **scheme's win/defeat consequence** — the reward for defeating the scheme-Villain — sitting at the same logical level as a Mastermind Tactic's defeat reward or a scheme's Evil-Wins clause, not at the level of a card's printed Fight effect.
- **OPTIONAL, NOT FORCED (the safety valve):** The Ultimate Nullifier is explicitly *"you MAY cancel that effect"* (inventory line 190, verbatim) — and Core p.7 SUPERPOWER/special-ability framing treats "you may" as the player's choice. So even under the current over-broad code, **no rational solo player would ever negate their own win.** A FORCED negate would be a real bug (it would cancel the player's win against their will); an OPTIONAL one is at worst a meaningless menu prompt. This downgrades the issue from "rules bug that can cost a win" to "cosmetic over-scoping." Still, exempting the win-trigger is the rules-correct fix and removes a confusing/nonsensical prompt.
- **RE THE GP-6b "BROAD" PREMISE:** The dispatch said docs/rules-notes/revelations.md already holds a "GP-6b BROAD" decision that the Ultimate Nullifier can cancel ANY Location's own Fight effect (pure Locations included). **That entry is NOT present in this file as of 2026-06-03** (file had only the two Korvac entries above; "GP-6b" / "Ultimate Nullifier" appear nowhere in docs/rules-notes/ — only in docs/card-effect-audit-results-2026-03-31.md line 87, which merely notes the negation is wired via `promptNegateFightEffectWithMrFantastic`). So the GP-6b BROAD ruling is either uncached or lives elsewhere. **Even if GP-6b BROAD is correct for genuine LOCATION FIGHT EFFECTS, it does NOT extend to the Korvac win-trigger:** GP-6b (as described) concerns a *Location's own Fight effect* — a Fight effect that genuinely fires "prior to comparing Attack totals" when you fight that Location. Korvac's "if you defeat" KO-Mastermind clause is NOT a Location Fight effect; it is a SCHEME on-defeat win-consequence that the digital build merely *reuses the Location `fightEffect` plumbing* to fire. The rules category is set by the card text + Core p.13 timing, not by which code field the implementation borrowed. The Location-reuse is an engine convenience and must not leak its "this is a fightEffect" status into the rules-facing negation scope.

Korvac win-trigger — inventory docs/card-inventory/final/revelations.md line 689 (verbatim: "This Scheme counts as a 19 Attack 'Korvac' Villain worth 9VP.
