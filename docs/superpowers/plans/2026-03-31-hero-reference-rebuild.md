# Hero Reference Data Rebuild — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild the hero sections of `docs/card-effects-reference/core-set.md` using `cardDatabase.js` as the authoritative source for all structured fields, with card images used only for effect text.

**Architecture:** A Node.js extraction script reads `cardDatabase.js` via the `vm` module, derives all structured fields, and outputs a pre-filled markdown draft and a metadata JSON. A subagent then reads each card image for effect text only, cross-checks any `[Tag]:` trigger patterns against the DB condition value, and flags mismatches for manual review. Once the "Needs Review" list is cleared, the draft replaces the hero section in the main reference file.

**Tech Stack:** Node.js (built-in `vm` and `fs` modules only — no npm), static `.webp` card images, markdown

### Task 1: Create the extraction script

**Files:**
- Create: `scripts/extract-hero-data.js`

- [ ] **Step 1: Create the scripts directory and write the script**

Create `scripts/extract-hero-data.js` with this exact content:

```javascript
const vm = require('vm');
const fs = require('fs');
const path = require('path');

// Load the database file
const dbPath = path.join(
  'Legendary-Solo-Play-main',
  'Legendary-Solo-Play-main',
  'cardDatabase.js'
);
let code = fs.readFileSync(dbPath, 'utf8');

// IMPORTANT: In Node.js vm, `const` declarations are block-scoped to the
// script and NOT accessible on the context object. We replace just the
// `const heroes = ` declaration with a plain assignment so it becomes
// available as context.heroes after execution.
code = code.replace(/\bconst heroes\s*=/, 'heroes =');

const context = {};
vm.createContext(context);
vm.runInContext(code, context);

const heroes = context.heroes;
if (!heroes) {
  console.error('ERROR: heroes array not found in cardDatabase.js');
  process.exit(1);
}

// Filter to Core Set heroes only (identified by image path)
const coreSetHeroes = heroes.filter(h =>
  h.cards && h.cards.some(c => c.image && c.image.includes('Reskinned Core'))
);

// Derive the base value display string from DB fields
function baseValue(card) {
  const hasAttack = card.attackIcon === true;
  const hasRecruit = card.recruitIcon === true;
  if (!hasAttack && !hasRecruit) return '0';
  const parts = [];
  if (hasAttack) {
    parts.push(card.bonusAttack > 0 ? '0+ [Attack]' : `${card.attack} [Attack]`);
  }
  if (hasRecruit) {
    parts.push(card.bonusRecruit > 0 ? '0+ [Recruit]' : `${card.recruit} [Recruit]`);
  }
  return parts.join(', ');
}

// Build markdown draft and metadata JSON
let markdown = `# Core Set Heroes — Draft\n\n`;
markdown += `<!-- Generated by scripts/extract-hero-data.js -->\n`;
markdown += `<!-- Structured fields from cardDatabase.js. Effect Text column filled in separately. -->\n\n`;
const metadata = {};

for (const hero of coreSetHeroes) {
  markdown += `### ${hero.name}\n\n`;
  markdown += `| Card Name | Class | Team | Cost | Base Value | Effect Text |\n`;
  markdown += `|-----------|-------|------|------|------------|-------------|\n`;

  for (const card of hero.cards) {
    // Strip hero name prefix from card name (e.g. "Black Widow - Mission Accomplished" -> "Mission Accomplished")
    const cardName = card.name.startsWith(`${hero.name} - `)
      ? card.name.slice(hero.name.length + 3)
      : card.name;

    const cls = (card.classes && card.classes.length > 0) ? card.classes[0] : 'None';
    const team = card.team || 'None';
    const cost = card.cost;
    const bv = baseValue(card);

    markdown += `| ${cardName} | ${cls} | ${team} | ${cost} | ${bv} | |\n`;

    metadata[card.name] = {
      image: card.image || '',
      condition: card.condition || 'None',
      conditionType: card.conditionType || 'None'
    };
  }

  markdown += '\n';
}

// Write outputs
const draftPath = path.join('docs', 'card-effects-reference', 'core-set-heroes-draft.md');
const metaPath = path.join('docs', 'card-effects-reference', 'core-set-metadata.json');

fs.writeFileSync(draftPath, markdown);
fs.writeFileSync(metaPath, JSON.stringify(metadata, null, 2));

console.log(`Done. ${coreSetHeroes.length} heroes extracted.`);
console.log(`Draft: ${draftPath}`);
console.log(`Metadata: ${metaPath}`);
```

**Subagent prompt:**

```
You are filling in the Effect Text column for Core Set hero cards in the Marvel Legendary game.

You have two input files:
1. `docs/card-effects-reference/core-set-heroes-draft.md` — a pre-filled table of all Core Set heroes. Every column is already correct EXCEPT the Effect Text column, which is blank.
2. `docs/card-effects-reference/core-set-metadata.json` — a JSON file with per-card metadata: image path, DB condition value, and conditionType.

Your job: for each card, read the card image and fill in the Effect Text column. Then write the completed table back to `docs/card-effects-reference/core-set-heroes-draft.md`.

## Card image layout — CRITICAL

Hero cards have this layout:
- Upper left (TOP): Team affiliation icon
- Upper left (BOTTOM, directly below team): Class icon
- Lower left: Base value (attack or recruit points when played)
- Lower right: Cost (recruit points to acquire)
- CENTRE: Effect text

You must ONLY read the CENTRE TEXT AREA of each card. Do not read any corner values.

## How to read effect text

1. Read the centre text area of the card image.
2. Replace any icon symbols with their text equivalents in square brackets:
   - Attack sword icon → [Attack]
   - Recruit star icon → [Recruit]
   - Covert (eye) icon → [Covert]
   - Strength (fist) icon → [Strength]
   - Instinct (lightning bolt) icon → [Instinct]
   - Ranged (crosshair) icon → [Ranged]
   - Tech (gear) icon → [Tech]
   - X-Men icon → [X-Men]
   - Avengers icon → [Avengers]
   - S.H.I.E.L.D. icon → [S.H.I.E.L.D.]
   - (other team icons similarly)
3. Write the full effect text in quotes, e.g. `"[Covert]: You may KO a card from your hand or discard pile."`

## Condition cross-check — CRITICAL

Some cards have a trigger condition: text like `[Covert]:` or `[Avengers]:` at the START of the effect that gates the rest of the ability. This is different from a class/team reference that appears in the BODY of the text.

The rule: if the effect text starts with `[Tag]:` (a bracketed tag immediately followed by a colon), that is a trigger condition. Compare that Tag against the `condition` value in the metadata JSON for that card.

- If they MATCH → write the effect text as-is.
- If they DO NOT MATCH → do NOT pick one. Add the card to a "## Needs Review" section at the bottom with both values shown, and write the effect text as you read it from the image (so there is something to compare against).

Cards with `condition: "None"` in the metadata: if the effect text starts with `[Tag]:`, add to "Needs Review" — this is unexpected and needs checking.

## Silver Surfer exception

Silver Surfer has no team affiliation. His cards will have an empty team slot in the upper-left-top position. This is correct — the Team column in his rows should stay "None". Do not flag this as an issue.

## Needs Review format

```
## Needs Review

The following cards had a mismatch between the effect text read from the image
and the DB condition field. Please check the physical card and correct the entry above.

- **[Hero Name] - [Card Name]**: image text starts with `[X]:` but DB condition is `Y`
```

## What to do

1. Read `docs/card-effects-reference/core-set-heroes-draft.md`
2. Read `docs/card-effects-reference/core-set-metadata.json`
3. For each card row in the draft, read the card image at the path given in the metadata JSON
4. Fill in the Effect Text column
5. Write the completed table back to `docs/card-effects-reference/core-set-heroes-draft.md`, appending a "## Needs Review" section at the bottom (or noting "None" if there are no mismatches)
```
